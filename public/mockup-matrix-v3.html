Sheet behavior: slides in from right over the matrix. Backdrop dims the rest. Favorites sorted first by default, then non-favorites. Search filters the whole list, not just visible. Sort/filter chips mirror the Recipes index pattern but compact (All / Favorites only / Recent). Width 460px — wide enough for full names + macros, narrow enough to keep some matrix visible on the left as context.

Mobile model

One day at a time. Day pills horizontal at top (the existing planner mobile pattern). Tap a pill to switch day.

Slot rows full-width. Breakfast / Lunch / Dinner stacked vertically. Each row shows recipe name, macros, ★ if favorited. Tap an empty slot to open the bottom-sheet picker.

"Need more?" footer exposes Side / Dessert / Beverage chips. Adding one inserts a slot row above. Snack is shown by default if any slot in the week has one (global pattern carries over).

Day totals at the bottom — 8 metrics in a 4-col grid. Three-state color same as desktop.

Pl day-pill dots indicate days with meals (solid) vs empty (hollow). Quick scan of week status.

Bottom sheet — same pattern as the existing AddMealSheet. Handle, head with eyebrow + search, segmented toggles (Favorites / All / Recent), scrollable list. Tap a recipe to fill the slot and dismiss the sheet.

The toggle order matters: Favorites first (default tab), All for the escape, Recent for "I cooked that last week and liked it." Recent should probably ship in phase 2 if we don't have the tracking infra yet.

04
What about the nutrition pull-out + swap?

The current planner's nutrition pull-out is two things at once: (1) a per-day totals view, (2) a "click a meal to see what to swap to hit goals" affordance. Matrix already does (1) better — totals live below every day column, always visible, no toggle needed. What's lost is (2): the swap suggestion.

Replacing swap with Claude via MCP: add two MCP tools — get_meal_plan_week(planId) and swap_meal(mealLogId, recipeId). In the Optimization prompt block (already exists on recipes), add a planner variant: "Wednesday is over on sat fat and sodium. What can I swap?" Claude reads the structured plan, the per-recipe nutrition, the user's goals, and the favorited candidates — then suggests swaps with rationale.

This is actually stronger than the current swap UI — the swap tool today shows you alternatives by nutrient, not by your goals as a whole. Claude with full context can balance everything at once. Phase 2. Not blocking matrix shipping.

For now (Phase 1): the three-state color on the totals row tells you which days are over. You click an over-limit cell and pick something different from your favorites. Manual swap. Good enough to ship.
