Hover behavior: the ⋯ button is invisible by default. Hover the day header → fades in to 100% opacity. Click → menu opens, button stays visible.

Two sections in the menu: Save (this day → template) and Apply template (existing templates → this day). The clean separation lets the same trigger handle both directions without modal switching.

Apply list is sorted most-recent-first. Item count shown on the right for context.

"Manage templates →" at the bottom of the menu opens the manage sheet (§04).

02
Save dialog
§ TRIGGERED BY "SAVE THIS DAY AS TEMPLATE…"
§ SAVE TEMPLATE

Save Wednesday as a template.

Reusable on any future day. The current servings, recipes, and pantry items are captured.

Capturing 5 items: 1 breakfast, 1 lunch, 1 dinner, 1 snack, 1 beverage.

Empty day: if the day has zero items, the "Save this day as template…" menu item is disabled with a hint "Add meals first".

Duplicate name: if the name already exists, submit shows an inline error "A template called 'Workout day' already exists. Rename or overwrite?" with Overwrite / Cancel buttons.

03
Apply confirm — target day has meals
§ APPLYING TO A DAY THAT ALREADY HAS MEALS · USES dialog.confirm
§ APPLY TEMPLATE

Replace Wednesday's meals?

Wednesday already has 4 meals. Applying "Workout day" (5 items) will replace them. Or append to keep both.

Three options: Replace deletes existing meals and creates new ones from the template. Append adds template items on top of what's there (could result in 2 breakfasts, etc.). Cancel closes without changes.

Empty target: if the day has no meals, no confirm needed — apply directly + toast "Applied 'Workout day' to Wednesday".

Mobile placement — no new chrome

⋯ added to the existing toolbar, right after + NEW. The button is 30×30 with a subtle outline. It refers to the active day (whichever pill is highlighted in the day strip).

Tap → bottom sheet. Reuses the matrix's existing iOS-style sheet styling (handle, rounded corners). Same Save / Apply structure as desktop. Switching days re-targets the menu's "this day" without closing.

No active-day tile. The accent-tinted day pill already communicates "you're looking at this day" — the heavy tile in v1 was duplicating that signal.

Meal count moves into the bottom sheet header ("Tuesday · 3 meals") rather than being shown inline above the slots. Quieter.

06
Open questions
Resolved

Apply default: Replace, primary button, black (not red). Red is reserved for true destructive actions (delete account, delete plan).
Templates household-scoped + tagged with personId. Manage list shows a small attribution chip per row ("From Jen", color dot). Future use: filtering, MCP-aware scaling suggestions.
Servings preserved. No rescaling on apply.
Missing recipe on apply: silent skip, toast "Applied 'Workout day' — 1 item skipped (recipe deleted)."
Search: include from v1. Small input above the list in the manage sheet. Above the apply menu when count > 6.
Mobile trigger: small ⋯ button in the existing toolbar, no new chrome.

Remaining design call

Empty day handling. If the user opens ⋯ on a day with zero meals, do we:
(a) Disable "Save this day as template…" with hint "Add meals first," OR
(b) Hide the Save section entirely so the menu only shows Apply?
Leaning (a) — visible affordance + explanation is better discovery than a missing menu.
