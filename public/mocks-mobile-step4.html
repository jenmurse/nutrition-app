Notes on the pattern. The top bar is a single 50px row with border-bottom: 1px solid var(--rule). Wordmark left, contextual right slot. On flow screens (Add Meal step 2) the wordmark is replaced by the back link — the wordmark's job is "you are inside the app" and a back link in a flow already says that more specifically. The current "section name" right slot from the bottom rail (PLANNER, RECIPES, PANTRY, SETTINGS) goes away — the page content already tells you where you are. Person chips on the dashboard live next to MENU on the right. On planner, the date range / person picker / + NEW PLAN row stays as a sub-toolbar below the top bar — same as today. The bottom rail is removed; menu sheet is now triggered from the top right.

Notes on the change. Each ingredient is a vertical block with three rows: (1) drag handle + name + delete, (2) amount + unit indented under the name, (3) preparation indented under the name. The drag handle and delete are aligned on row 1 only — the indent on rows 2 and 3 visually anchors all sub-fields to the ingredient name above. Every field has a 9px DM Mono uppercase label above it (Amount, Unit, Preparation) — this lifts the prep field out of placeholder-only land and matches the labeling pattern used elsewhere in the app. Delete is a 32x32 tap target with a hairline border, matching the pantry list edit/delete pattern. The select uses the locked bottom-border-only style with a custom caret (per Brief 2D). On desktop, the same component can revert to the current 4-column row at min-width: 641px — the new stacked layout is mobile-only.
