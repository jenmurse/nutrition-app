§ INDEX PAGES · MOCKUP V1

Four index pages, re-registered to match the landing.

No page headers, no cards, no pill buttons. Toolbars reduced to text chips with hairline underlines on active. Count in mono, sharp primary button anchors the right edge. Nutrition values live in a 4-stop mono row on list views (kcal · fat · carbs · prot), and as a full 8-row block on the pantry grid — the only view where the full nutrition card earns its space.

Three layout rules to scan against: containers sharp (grid cells are hairline-ruled, no boundaries, no shadows); selectors hairline (toolbar chips, grid/list, sort — text with underline on active); identity markers round (avatar dots only, unchanged).

Desktop only · filter chip row scrolls horizontally when crowded · mobile pass separate

Rule set
Containers sharp. Buttons, inputs, cards, modals, dropdowns — zero border-radius. This is the load-bearing change. Selectors hairline. Filter chips, view toggles, sort triggers — no borders, active state is a 1.5px underline. Reads as typography, not chrome. Identity markers stay round. Avatar dots, person-switcher pills (JEN/GARTH/EVERYONE), theme dots, checkboxes, radios, icon-only buttons. Roundness signals identity or touch, not container.
