What changes, what stays
Stays the same

Editorial dashboard chrome. Hero greeting, big "Good morning, Jen", date eyebrow, person switching, getting-started card. The matrix-aligned slot stack only replaces the "Today's key meals" block below.

The data. Same source — today's MealLogs from the active person's plan. Same numbers as the matrix's "today" column.

"Open planner →" link. Stays. Now points to /planner with today's planId.

What changes

Editorial 3-column cards → matrix-style slot rows. Each meal is one row: slot label (mono) · name (sans 20px 600) + 1-line meta · 2–3 macros chip on the right. Identical to the matrix mobile day view.

Per-meal "See recipe →" link drops. The slot row is the click target — taps into the planner cell or recipe detail. Reduces visual noise.

"Today's totals" strip appears below. The 8-cell totals grid from the matrix (Cal / Fat / Sat F / Na / Carb / Sugar / Prot / Fiber) with three-state color. Gives the dashboard a clear "how am I doing" answer that the editorial cards don't provide.

Why this is better

Visual continuity. A user lands on the dashboard, sees the same row pattern they'll see in the matrix. No mental gear-shift between "summary view" and "planner view" — they're the same vocabulary at different densities.

Goal feedback is at-a-glance. The current cards show per-meal stats but not how the day is pacing toward goals. The totals strip with color does.

Shows all meals, not just three. Editorial filters out empty types; this stack shows Breakfast / Lunch / Dinner + any extras (Snack, Side, Dessert, Beverage). True picture of the day.

Open question

Do we keep the editorial columns anywhere? The big numbered "01 · BREAKFAST" feels good on first install when there's room to breathe. Maybe a setting? Or just retire it — the matrix is the editorial moment now.
