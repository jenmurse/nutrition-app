Planner zoom-out

Three ways to show "more than one week at a glance," ranked from least to most disruptive. Each shows the same data: 4–5 weeks of planning state. Click any day/week to jump the matrix to it.

A — 30-day strip above the matrix

Recommended

A 30-day sparkline that lives above the existing matrix. Each bar's height = how much of the day is planned (slot fill %). Current week is highlighted in green. Click any day → matrix jumps to that week. The matrix stays the source of truth; this is just a small navigation + density hint.

+ Pros

  • Doesn't replace the matrix — pure additive
  • Cells small enough that "density at a glance" is the right reduction
  • Works on mobile (horizontal scroll if needed)
  • Shows "I haven't planned far ahead" at a glance

− Cons

  • Doesn't tell you what's on each day, only how full
  • Some users may not see what the bar heights mean without a legend

B — Month grid

Reference / probably no

A literal calendar month. Each cell shows the dinner name + dots for filled slots. Current week is highlighted. Click any day → matrix jumps to that week. Shown here so we can see what's lost.

+ Pros

  • Familiar mental model (everyone knows calendars)
  • Shows dinner name → "have I cooked this recently?"

− Cons

  • Cells too cramped on mobile to read meal names
  • Replaces the matrix or adds a second navigation surface that competes with it
  • Lunch/breakfast/snacks invisible — only dinner fits
  • Maintaining two layouts that show overlapping data is fragile

C — "Coming up" week cards

Alternative

A vertical list of upcoming weeks. Each card shows the week range, dinner names, and a fill %. Click → jump matrix to that week. No calendar grid — reads like an editorial list.

+ Pros

  • Reads great on mobile (vertical list, no grid math)
  • Shows actual meal names — answers "have I cooked X recently?"
  • Editorial tone fits the rest of the app

− Cons

  • Not really a "zoom out" — more like a week index
  • Adds a third surface (matrix, browse, this) that you have to maintain
  • Hard to fit on desktop without feeling sparse

My take

Build A. It's small, additive, doesn't fork the matrix as the primary surface, and gives a real signal (where are the planning gaps?) without faking that we can fit five weeks of meal names on one screen. C is good but solves a different problem ("what did I plan?") that we can revisit later. B is the option that sounds right but doesn't survive contact with real cell sizes.
