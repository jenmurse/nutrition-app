The slot model: A "Breakfast slot" is a typed slot that holds breakfast. That's what filters the picker to only show breakfast-tagged favorites. Generic slots would lose the filtering, which is the whole point. Default = 3 typed slots (B/L/D). The "Need more?" row exposes typed additions — Snack / Side / Dessert / Beverage / Custom. Slot labels are draggable to reorder (the ⋮⋮ grip).

Pros: Strict grid alignment. Easy comparison across days. The "+ pick" affordances nudge consistency.
Cons: 7-day snack row when you only want one snack on Tuesday feels heavy. Lots of empty cells.

B · Per-day stacks
Add a Snack only on Tuesday → only Tuesday gets a snack cell. Each day column has its own height. More honest to how people eat — most days don't get a snack. The slot label column stays for B/L/D rows, but day-specific additions append to each column's bottom.

My recommendation: Approach A · Global rows. The strict grid is a load-bearing part of the aesthetic — it's what made you say the matrix matches the rest of the app and the planner doesn't. The empty "+ pick" cells in the snack row aren't waste — they're invitations. If your snack pattern is "only on workout days," you fill some and leave others. The row is a recurring question, not a demand. And the rare case where you want one truly one-off addition (cocktail on Friday night) can use the "+ Custom" affordance from the "Need more?" footer row, which we can decide later whether to make per-day or global.

03
Adding slots — the "Need more?" affordance
§ TYPED ADDITIONS · NEW SLOT TYPE APPEARS AS A ROW

The bottom row of the matrix has the "Need more?" affordance with explicit chips for each typed addition. Clicking + Snack adds a Snack row to the matrix (global, Approach A) above the "Need more?" row. The order: B / L / D / [user additions in add-order] / "Need more?". Once a row is added, you can drag-reorder via the slot label.

Need more?

Removing a slot: hover the slot label → small × appears next to ⋮⋮. Click to remove the row. If the row has any filled cells, confirm dialog: "Remove the Snack row? 3 snacks across this week will be deleted."

"+ Custom" affordance: for one-off rows. Opens a small dialog: name the row (e.g. "Late-night tea") + pick a meal-type for picker filtering. Probably defer to phase 2.

04
Reordering slots
§ DRAG THE ⋮⋮ GRIP ON THE SLOT LABEL

Why reorder? If you eat dinner early and have a snack later, you might want Breakfast / Lunch / Snack / Dinner. Drag the ⋮⋮ grip on the slot label to reorder. Drop targets show a 2px black bar between rows.

Slot order is per-plan, not global. So different weeks can have different orderings (workout week vs travel week). Maybe later we add saved orderings as part of "Saved day templates." Phase 3.
