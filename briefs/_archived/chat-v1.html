§ Mock · v1

Ask — in-app AI chat surface.

Document-style messages, no bubbles. Propose-then-confirm for all writes — confirm-cards use the same ruled-block + ruled-row vocabulary as the rest of the app, so a proposed meal swap looks like a real planner row, not a chatbot suggestion. Two presentation pairs: a right-docked panel on desktop, a bottom sheet on mobile. Shared internals.

01Trigger affordances

How you open it from the global nav.

Same glyph on both surfaces — ✦, sized to match the existing settings cog on desktop and the hamburger on mobile. Placed before the person dots on desktop, between the person pulldown and the hamburger on mobile. The glyph picks up the active person's accent color, a small identity signal consistent with the existing person-as-accent convention. The label "Ask AI" appears in the panel header once open — earning the description space where it actually helps, not crowding the nav.

02Desktop · empty state

What you see the first time.

Short editorial lede plus two example prompts as ruled-block lines. No "suggested prompts" chips below the input — that's chat-app vocabulary we don't speak.

Plan a day, dial in a recipe, or check tomorrow's groceries. Ask in plain language — you'll see the proposed change before anything is saved.

"Fill next week with high-protein dinners."

03Desktop · read-only answer

Answers read as paragraphs, not bubbles.

Mono speaker label above each message. YOU in the active person's accent (identity — this is whose voice it is), GOOD MEASURE in ink (the system, constant across users). Body in DM Sans 13px, var(--fg-2), with key numbers in ink + 600 weight so they land as data, not prose.

04Desktop · single write

Proposed change as a real artifact.

The confirm-card uses the planner's own ruled-row vocabulary — eyebrows, item names, mono macros, sharp borders. From → arrow → To for swaps. Macro delta row shows what actually changes; accent green for positive deltas, semantic red for negative on capped nutrients. CANCEL ghost mono, APPLY filled black — one primary commit per card.

06Desktop · bulk write

A whole week, as one card.

Same card shape; the body is just longer. Per-day ruled rows. Summary macros at the bottom show the week-wide deltas vs. the user's targets. v1 ships all-or-nothing CANCEL / APPLY. A future v2 could add per-row "Skip" affordances; the data model and API are ready for it from day one.

09Type & treatment spec

Locked, for the build.

Mirrors the existing design-system rules — no new tokens, no new fonts. Production classes will use the ck- prefix (chat-kit), scoped under the panel / sheet container so nothing leaks into the rest of the app.
