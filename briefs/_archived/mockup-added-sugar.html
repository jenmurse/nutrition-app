Added sugar tracking — UI study

Two surfaces benefit from a quick design pass before coding: the daily totals strip on the matrix planner (which gains a 9th row), and the recipe nutrition panel (which introduces the new "unknown" state — the first time a value renders as — instead of 0). Other surfaces (pantry form input, Settings goals layout, Dashboard stats) are mechanical and inherit from the patterns established here.

§ 01 Matrix daily totals — 9-row variant

Adding Added Sugar bumps the strip from 8 rows to 9. Stacked vertically, this is fine — no horizontal layout question. The decisions are placement (where in the order) and unknown-state visual (when one ingredient on the day lacks data).

§ 02 Recipe nutrition panel — the unknown state

This is the first place in the app where a missing value renders as — instead of 0. The visual decision is how to read the unknown clearly without making it feel like an error. Two scenarios shown: a recipe where added sugar is known (built from ingredients that all have data), and one where it's unknown (at least one ingredient lacks added-sugar data).

The unknown state visual uses — at color: var(--muted) with no opacity reduction — distinct from the "empty" state (which uses opacity 0.4 for cells with no value entered at all). Goal `/ 6g` still renders so the user sees the target they're missing.

The contextual hint uses the existing margin-note pattern (left-rule, no fill, 11px), with the count of ingredients responsible. The hint is scoped to this nutrient — not a banner — because only the added-sugar row is affected.

Editorial copy: "Added-sugar total unknown" reads as factual, not error-like. Tells the user what's missing and what they can do about it.

§ 03 Decisions to lock in

Before code:

  • Placement: Variant A — Added Sugar paired with Sugar.
  • Label: "Added Sug" in 9px mono at narrow widths; "Added Sugar" anywhere with room.
  • Unknown render: — in var(--muted), no opacity adjustment. Distinct from empty (0.4 opacity).
  • Goal still renders next to — so the user sees the target.
  • Contextual hint appears only on the recipe nutrition panel, only when the value is unknown. Daily totals don't get a hint — the value alone is enough signal.
  • Color rule: standard three-state (over=red, met=green, neutral=fg). Plus a fourth "unknown" mapping to muted. No green-met state for added sugar (it's a cap, not a minimum).
