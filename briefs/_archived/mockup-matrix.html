03
Open questions
A · Slot rows

Default to 4 slots (Breakfast / Lunch / Dinner / Snack). Side / Dessert / Beverage are hidden by default. Add a small "+ Add slot" control at the bottom of the slot column to expose the others when needed. Alternative: show all 7 always — but the matrix gets noisy and most days don't have a dessert.

B · Picker contents

First page = favorited recipes filtered by slot tag. Footer link "Browse all lunch recipes →" opens a larger sheet with the full library if you want something outside favorites. Keeps the curated experience but doesn't trap you.

C · Totals shown

In the filled state above each column gets the full 8-metric stack. Tight column widths compress the labels but keep all the data. If this feels too dense we can collapse to Cal · Prot · Fiber only by default and let the user expand to all 8 — but I think the full stack is the right call. It's the whole point of matrix mode.

D · Commit semantics

"Commit week" replaces the existing plan's meals for that week. Confirmation dialog: "Replace this week's plan with 28 matrix meals?" Alternative: append (add matrix meals on top of existing) — but this gets messy fast. Replace is cleaner.

E · Per-person matrix

Matrix runs against the active person's favorites. Switching person chips re-populates the picker. EVERYONE mode is a Phase 3 question (probably shows only recipes favorited by all selected people).

F · Empty favorites in a slot

If a slot has zero matching favorites, the picker shows: "No favorited lunch recipes." with one link: "Browse all lunch recipes →". Don't fall back to showing everything — that defeats the curation layer.

G · Mobile

Phase 2 problem per the brief. On iPad it works fine on the desktop layout. iPhone gets deferred.
